<h2 align="center"><b> What is Methods ?</b></h2>

* Java Methods are blocks of code that perform a specific task. 
* A method allows us to reuse code, improving both efficiency and organization.

<h4 align="center"><b>ðŸ”¹ Method Syntax</b></h4>

```
returnType methodName(parameters)
```

<h5><b> Example </b></h5>


```
int add(int a, int b) {
    return a + b;
}
```

<h5><b> Parts </b></h5>

* ```returnType``` â†’ type of value returned
* ```methodName``` â†’ name of the method
* ```parameters ```â†’ inputs to the method

 <h2 align="center"><b> ðŸ”¹ Void and Non-Void Methods </b></h2>
 
<h4 align="center"><b> âœ… Void Method </b></h4>

* Does <b>not return any value</b>
* Used to perform an action
```

    System.out.println("Hello");
}
```

<h4 align="center"><b> âœ… Non-Void Method </b></h4>

* <b> Returns a value</b>
* `return` statement is mandatory
  
```

    return n * n;
}
```

<h2 align="center"><b> ðŸ”¹ Parameters: Formal vs Actual </b></h2>
<h4 align="center"><b> âœ… Formal Parameters </b></h4>

* Defined <b> in the method definition  </b>
* Receive values when the method is called
* Act like <b> local variables </b>
  
```
void add(int a, int b) {   // a, b â†’ formal parameters
    System.out.println(a + b);
}
```

 <h4 align="center"><b>âœ… Actual Parameters (Arguments) </b></h4>

* Passed <b> during method call </b>
* Provide real values to the method
```
add(5, 3);   // 5, 3 â†’ actual parameters
```
